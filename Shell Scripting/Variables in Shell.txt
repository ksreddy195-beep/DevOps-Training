variables in shell scripting:

containers that store values like text, numbers, or command output.

Example:
1.Creating variables:
Syntax:
name="satish"
    age=31
 
Rules: No spaces around =

To access created variables:
    echo $name
------------------------------------------------------------------------------------------
2. Using Variables: The name of a variable can contain only letters (a to z or A to Z), numbers ( 0 to 9) or the underscore character ( _).
#!/bin/bash

course="Shell Scripting"
echo "Learning $course"
---

3.Valid and Invalid Variables:
The following examples are valid variable names −

_ALI
TOKEN_A
VAR_1
VAR_2
Following are the examples of invalid variable names −

2_VAR
-VARIABLE
VAR1-VAR2
VAR_A!

The reason you cannot use other characters such as !, *, or - is that these characters have a special meaning for the shell.
---
4. Read only Variables:
Shell provides a way to mark variables as read-only by using the read-only command. After a variable is marked read-only, its value cannot be changed.

For example, the following script generates an error while trying to change the value of NAME −

#!/bin/sh

NAME="Zara Ali"
readonly NAME
NAME="Qadiri"

---
5. Unsetting Variables:
Unsetting or deleting a variable directs the shell to remove the variable from the list of variables that it tracks. Once you unset a variable, you cannot access the stored value in the variable.

Following is the syntax to unset a defined variable using the unset command −

unset variable_name
The above command unsets the value of a defined variable. Here is a simple example that demonstrates how the command works −

#!/bin/sh

NAME="Zara Ali"
unset NAME
echo $NAME
---
6.Variable Types:
When a shell is running, three main types of variables are present −

1. Local Variables

Exist only inside the current shell session.

Not passed to child processes (programs started by that shell).

Created at the command prompt or inside scripts.

Used for temporary values.

Example:

name="Sandeep"

2. Environment Variables

Available to the shell and all child processes.

Needed by many programs to run correctly.

To convert a local variable into environment variable, you use export.

Example:

export PATH
export JAVA_HOME="/usr/lib/jvm/java-11"


Common environment variables:

$PATH

$HOME

$USER

$SHELL

3. Shell Variables

Special variables created and used by the shell itself.

They control how the shell behaves.

Some are local, some are environment variables.

Examples:

$PS1 → prompt format

$UID → current user ID

$PWD → current directory

$0 → script name

$# → number of arguments
